# Customer_behaviour_analysis
# 🛍️ Retail Customer Behavior Analysis: End-to-End Data Analytics Project

## Overview

This project demonstrates an end-to-end data analytics workflow, transforming raw retail customer data into actionable business insights. The objective was to understand customer shopping behavior, purchasing patterns, and factors influencing sales performance, customer satisfaction, and loyalty.

The project follows a complete analytics lifecycle, including data cleaning and exploration with Python, business analysis using SQL, and interactive dashboard creation in Power BI.

---

## Business Problem

Retail businesses generate large amounts of customer transaction data, but without proper analysis, valuable insights remain hidden. This project aims to answer key business questions such as:

* Which customer demographics generate the most revenue?
* Do subscribed customers spend more than non-subscribers?
* Which products receive the highest customer ratings?
* How do discounts influence purchasing behavior?
* Which customer segments are most valuable to the business?

The insights generated can help businesses improve marketing strategies, customer retention initiatives, and product decision-making.

---

## Dataset

The dataset contains retail customer information, including:

* Customer demographics (Age, Gender)
* Product categories and purchased items
* Purchase amounts
* Subscription status
* Shipping preferences
* Discount usage
* Review ratings
* Previous purchase history
* Purchase frequency

## Project Workflow

### 1. Data Loading

* Loaded the raw retail dataset into Python using Pandas.
* Inspected dataset structure and data types.
* Checked for missing values and inconsistencies.

### 2. Exploratory Data Analysis (EDA)

Performed exploratory analysis to understand:

* Distribution of customer demographics
* Purchase behavior across categories
* Spending patterns
* Review ratings
* Subscription trends
* Discount usage patterns

### 3. Data Cleaning & Transformation

Data preparation steps included:

* Handling missing values using category-based median imputation for review ratings.
* Standardizing column names to snake_case format.
* Transforming categorical purchase frequency data into numerical values.
* Creating additional analytical features.
* Removing redundant and unnecessary columns.
* Validating data consistency and quality.

### 4. Database Integration

The cleaned dataset was loaded into a PostgreSQL database hosted on Supabase using SQLAlchemy and Psycopg2.

Workflow:

Raw CSV → Python Cleaning → PostgreSQL Database

This simulated a real-world ETL process where transformed data is loaded into a centralized database for analysis and reporting.

### 5. Business Analysis Using SQL

Several business questions were answered through SQL queries, including:

#### Revenue Analysis

* Total revenue generated by male vs female customers
* Revenue by age groups and product categories

#### Subscription Analysis

* Comparison of average spending and total revenue between subscribed and non-subscribed customers

#### Discount Analysis

* Products with the highest percentage of discounted purchases
* Customers who spent above average despite receiving discounts

#### Customer Segmentation

Customers were classified into:

* New Customers
* Returning Customers
* Loyal Customers

using Common Table Expressions (CTEs) and CASE statements.

#### Product Performance

* Top-rated products based on average review ratings
* Revenue performance across product categories
* Purchase patterns by shipping type

## Key Findings

* Revenue distribution between male and female customers can reveal high-value demographics.
* Subscription status does not necessarily guarantee higher customer spending.
* Certain products heavily rely on discounts to drive purchases.
* Loyal customers represent a significant opportunity for retention strategies.
* Customer review ratings provide valuable insights into product performance and customer satisfaction.
